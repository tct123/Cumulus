XCVario pairing

Cumulus can be paired via different interfaces with an XCVario device. Usually the interfaces Bluetooth or WiFi are used for this purpose. How to setup such a pairing, look into the user guide of the XCVario devive.

XCVario pairing via Bluetooth

You can connect Cumulus via Bluetooth to an XCVario and exchange data via the serial interface S1 of the XCVario device. Usually you will connect a Flarm device at this serial interface. For usage you have to pair Cumulus with the XCVario device. In the XCVario device you have to activate the routing of S1 via Bluetooth.

XCVario pairing via WiFi

Via the WiFi interface you can establish a connection to an XCVario device and exchange data via the interfaces S1, S2 and the internal data channel of the XCVario. There are three Tcp ports available under the IP address 192.168.4.1. A WiFi connection must be established from Cumulus to the XCVario device, where the XCVario devive is working as a WiFi server. The WiFi access point uses the name XCVario-[nnnn]. The related password is xcvario-21.

Note: These connections will only work properly, if the interfaces WLAN, BT, S1, S2 and the routing are setup correctly in the XCVario device. Have a look into the user guide of the XCVario device.

XCVario IP 192.168.4.1
Channel Port Description
WiFi-1 8880 XCVario transmits sensor data according to the configured NMEA protocol in the XCVario device.
WiFi-2 8881 FLARM at the serial interface S1-RS232. Cumulus evaluates these data and can also send commands to the Flarm device.
WiFi-KRT2 8882 KRT2 radio device at the serial interface S2-RS232. Cumulus can send frequencies to the radio to be setup as active or standby. The serial interface S2 of the XCVario device has to be configured as 9600 baud, 8 data bits, without parity.
